Communications Intelligence System · Nike

Nine packages. One operating system. Decisions that don't wait.

A two-layer system: intelligence that gathers, and a loop that turns it into better decisions.

The Nike communications function runs on talent and hustle. This system gives it memory, sensing, and a feedback loop — so every decision is made with more context and less prep time, and the function gets smarter every cycle.

The Challenge

Nike's comms function runs on talent and hustle.
But the system underneath is working against it.

Memory lives in people
Approved language, past positions, coverage archives — spread across drives, email, agency decks, and the one person who happens to remember.
Intel arrives too late
Competitive narrative shifts, cultural moments, policy signals — discovered after the window to respond has already closed.
Prep is manual heroics
Earnings Q&A, exec briefing books, stakeholder profiles — built by hand before every major moment, then discarded after.
Plans aren't stress-tested
The senior calendar can only hold a few pre-mortems. Most plans ship without a real adversarial read on how they'll land.

The team doesn't lack skill. The system lacks memory, sensing, and a feedback loop.

The Answer

A two-layer system: intelligence that gathers, and a loop that turns it into better decisions.

The Workshop Mechanic

Every package runs the same rhythm, so the session produces a comparable grid, not nine loose conversations.

Step One

Prioritize

Walk Nike through all nine packages in brief, then ask which matter most. Start where they point — so the session spends its time on the areas they care about and does not run out of clock.

Step Two

Pressure Test

For the packages Nike prioritizes, run the fixed question battery. The same five questions every time build rhythm and reveal which ones get hot.

  1. 1. Where does this happen today, and who owns it?
  2. 2. Last time it went wrong or too slow, what did it cost?
  3. 3. What information would this need that you have? That you do not?
  4. 4. What decision gets faster or better if this works?
  5. 5. What would make you trust the output enough to act on it?
Step Three

Reduce

Capture answers into five requirement categories — identical for every package, so results compare cleanly across all nine.

  • Inputs — data and access required
  • Owners — who runs it, who decides
  • Cadence — one-time, weekly, quarterly, always-on
  • Integration — where it meets existing workflow
  • Proof — what success looks like in 90 days
